## 🧩 Jieum

**Jieum**은 백석대학교 학생들을 위한  
**신뢰 기반 스터디 매칭 플랫폼**입니다.

교내 스터디나 공모전에 참여하고 싶어도  
함께할 팀원을 구하기 어렵고,  
익명 커뮤니티에서는 신뢰 부족으로 인해 실제 매칭으로 이어지기 어렵다는 문제를 발견했습니다.

👉 이를 해결하기 위해  
**사용자 정보와 활동 기반의 스터디 매칭 서비스**를 기획 및 개발했습니다.

---

## 🎯 Problem & Solution

### ❗ Problem
- 스터디/팀원 모집의 어려움  
- 익명 커뮤니티 기반 매칭의 낮은 신뢰도  
- 참여자의 성실도 및 협업 태도 파악 불가  

### 💡 Solution
- 스터디 모집 및 참여가 가능한 플랫폼 구축  
- 사용자 정보(학부, 관심사 등)를 기반으로 한 매칭  
- ‘스터디 온도’ 개념을 통해 신뢰도 판단 기준 제공  

👉 단순 모집이 아닌  
**신뢰 기반 협업 환경 제공**을 목표로 설계

## 📈 What I Learned

### 1️⃣ 상태 설계가 UI 구조를 결정한다는 것을 학습
댓글 기능 구현 과정에서 단순 배열이 아닌 아래와 같은 구조를 설계했습니다.

```js
{
  text: "",
  replies: []
}
```

처음에는 댓글과 답글을 분리된 구조로 관리하려 했지만,
구조가 복잡해지고 UI 렌더링 또한 비효율적이라는 문제를 발견했습니다.

👉 댓글 내부에 replies를 포함시키는 구조로 개선함으로써

댓글 / 답글 UI를 일관되게 렌더링할 수 있었고
기능 확장(답글 추가, 개수 계산 등)이 훨씬 수월해졌습니다.

➡️ 이를 통해 데이터 구조 설계가 곧 UI 설계와 직결된다는 것을 학습했습니다.



### 2️⃣ 작은 UX 처리 하나가 사용자 경험을 크게 바꾼다는 것을 경험

댓글 입력 시 아래와 같은 로직을 적용했습니다.

```
if (commentText.trim()) {
```

초기에는 공백만 입력해도 댓글이 등록되는 문제가 있었고,
이는 사용자 입장에서 불필요한 데이터와 혼란을 유발할 수 있었습니다.

👉 trim()을 활용해 공백 입력을 방지함으로써

잘못된 입력을 사전에 차단하고
보다 자연스러운 사용자 경험을 제공할 수 있었습니다.

➡️ 이를 통해 디테일한 입력 처리 하나가 UX 품질을 좌우한다는 점을 깨달았습니다.



### 3️⃣ 사용자 인지 기반 UI 설계의 중요성 이해
댓글에서 작성자를 구분하기 위해 다음과 같은 로직을 구현했습니다.

```
const isAuthor = userID === authorID;
```

이를 기반으로 작성자에게 “작성자” 뱃지를 표시했습니다.

👉 이 기능을 통해

사용자 간 관계를 직관적으로 파악할 수 있게 되었고
댓글의 신뢰성과 가독성이 향상되었습니다.

➡️ 이를 통해 단순 기능이 아닌 ‘사용자 인지’를 고려한 UI 설계의 중요성을 학습했습니다.



### 4️⃣ 사용자 액션에는 반드시 피드백이 필요하다는 점 학습
스터디 신청 기능에서 SweetAlert2를 활용하여
확인 모달을 추가했습니다.

```
Swal.fire({
  title: '신청 완료 하시겠습니까?',
})
```

초기에는 버튼 클릭 시 즉시 신청이 완료되었지만,
사용자 입장에서 의도하지 않은 행동이 발생할 가능성이 있었습니다.

👉 확인 모달을 추가함으로써

사용자 의도를 한 번 더 검증하고
행동에 대한 명확한 피드백을 제공했습니다.

➡️ 이를 통해 사용자 액션에는 반드시 명확한 피드백이 필요하다는 것을 학습했습니다.
